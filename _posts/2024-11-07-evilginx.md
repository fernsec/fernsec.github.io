Evilginx is a ready-made phishing kit. It's commonly used by threat actors for initial access ([T1566: Phishing](https://attack.mitre.org/techniques/T1566/)). It's an easy way in, as it's relatively simple to exploit a human (who is already inside), as opposed to exploiting a public-facing system which is likely secured and monitored.

Evilginx makes this easy with it's [open-source software model](https://github.com/kgretzky/evilginx2) and [accessible documentation](https://help.evilginx.com/). It even has a 5-min quick start guide!

In this article I aim to set it up, look at its capabilities, and test them with real victims. I will also look at whether there is any way to defend against these attacks, from a personal and enterprise perspective.

*The 'real victims' are my friends and they have consented to this.*

## My workstation

I am using Parrot OS 6.10.11 installed in VirtualBox for this workshop. As one should always do, I updated all my packages before starting.

```
sudo apt update && sudo apt upgrade
```

## First time setup

> The Evilginx quick start guide can be found [here](https://help.evilginx.com/docs/category/getting-started).

As an aspiring phisher, I wanted to get started straight away so I didn't build Evilginx from source. I simply downloaded the latest binary from [the project's GitHub](https://github.com/kgretzky/evilginx2/releases). This one was called "3.3.0 - Go & Phish".

I then followed the developer's 5-minute quick start guide, which I've linked above. It clearly sets out how to deploy a local installation, which is what I wanted to do to get familiar with the kit.

An important consideration is TLS certificates, which are very much necessary for a successful phishing campaign. In my case, this is not necessary as I will only be running a local installation. Self-signed certificates are sufficient which Evilginx will use if ran with the `-developer` flag.

The commands I ran are as follows:

```
# Run for first time to generate root CA certs

sudo ./evilginx -developer

# Copy certs to Linux cert store and install

sudo cp /root/.evilginx/crt/ca.crt /usr/local/share/ca-certificates/evilginx.crt
sudo update-ca-certificates
```

I then followed subsequent steps which include:

- Setting up domain
- Setting up phishlet
- Setting up lure

### Domain

I came up with **phishtagram.com**.

```
# Follow quick start guide commands
: config phishtagram.com
: config ipv4 127.0.0.1
```

I didn't officially register this domain, but I did (unnecessarily) add it to the file `/etc/hosts` which means that my local machine will resolve this domain to whatever IP I tell it to, overriding any DNS servers.

As you will see in the next section, phishlets make use of subdomains. **So there is no need to add the root domain to the hosts file yet.**

### Phishlets

As I understand it, these is the configuration file that tells Evilginx how to interact with the legitimate website and what to intercept.

These are tricky. Something that is not shown in the guide is that you need to acquire phishlets yourself.

You need to:
- program them yourself, or 
- download them from somewhere else.

I was only at the entrance of the Evilginx rabbit hole, so I thought that downloading a ready-made phishlet and hoping it worked made the most sense.

I did a quick google search for "instagram phishlets" which led me to [charlesbel's repository](https://github.com/charlesbel/Evilginx2-Phishlets/tree/main), containing the phishlet in question.

Once downloaded, I placed it in the `phishlets` folder.

```
└── evilginx-v3.3.0-linux-64bit
    ├── evilginx
    ├── phishlets
    │   ├── example.yaml
    │   └── instagram.yaml
    └── redirectors
        └── download_example
            └── index.html
```

Running the following in Evilginx got it working. This assigns this particular phishlet to any subdomain you want (so you could run multiple phishing campaigns at once from a single instance or domain).

```
: phishlets hostname instagram login.phishtagram.com
: phishlets enable instagram
```

I also added this subdomain to `/etc/hosts`.

```
# Get pre-configured list from Evilginx (handy command)
: phishlets get-hosts linkedin

# Open hosts file and paste in (I like nano)
sudo nano /etc/hosts
```

### Lures

The last step is to generate the phishing URL, or lure. This part is easy.

```
: lures create instagram
: lures get-url 0
```

### It works! (almost...)

Firefox warned me about bad certificates which is expected in my case, as I'm using self-signed certificates. But it worked!
